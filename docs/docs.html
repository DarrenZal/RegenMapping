Project Overview

The Regen Mapping project is a comprehensive knowledge graph and visualization platform for mapping regenerative organizations, people, and relationships across the globe.

Project Vision

The Regen Mapping project aims to create an interactive 3D globe and force-directed network graph that visualizes the regenerative economy ecosystem. By mapping organizations, individuals, and their relationships, we're building a living directory that helps discover regenerative organizations, connect complementary projects, visualize the network topology, track growth and evolution, and facilitate collaboration.

Lossless Conversion with JSON-LD @reverse Links

We've implemented a lossless round-trip conversion approach using JSON-LD @reverse links:

{
  "name": "Dylan Tull",
  "primary_url": "https://dylantull.com",
  "@id": "https://murmurations.network/profile/dylan-tull",
  "@reverse": {
    "schema:isBasedOn": {
      "@id": "https://raw.githubusercontent.com/DarrenZal/RegenMapping/main/profiles/unified/regen-person-dylan-tull.jsonld"
    }
  },
  "profile_source": "https://raw.githubusercontent.com/DarrenZal/RegenMapping/main/profiles/unified/regen-person-dylan-tull.jsonld"
}

How It Works:

  1. Unified ‚Üí Murmurations:
    • The lens transformation is applied
    • A JSON-LD @id field is added to identify the Murmurations profile
    • A JSON-LD @reverse link with schema:isBasedOn is added, pointing to the original unified profile
    • A profile_source field is added for backward compatibility
  2. Murmurations ‚Üí Unified:
    • First, the system checks for the @reverse link with schema:isBasedOn
    • If not found, it checks for the profile_source field
    • It fetches the original unified profile using the URL from either source
    • If the fetch fails, it falls back to using the lens transformation

Data Loss in Schema Transformations

When converting between schema formats using Cambria lenses, some data loss occurs due to the differences in schema structures:

Unified ‚Üí Murmurations (Source of Truth to Index Format)

This transformation is lossy because the Murmurations schema is simpler and doesn't support all fields in our unified schema:

  • ‚úÖ Basic fields like name, primary_url, locality, and tags are preserved
  • ‚úÖ Relationships are preserved
  • ‚úÖ Geolocation is preserved but restructured (latitude/longitude ‚Üí lat/lon)
  • ‚ùå Context information (@context, @type) is lost
  • ‚ùå Namespace prefixes are removed
  • ‚ùå Rich semantic structure is flattened

Murmurations ‚Üí Unified (Web App Recovery)

When converting back to unified format in the web app, we partially recover lost information:

  • ‚úÖ Context object is reconstructed with standard namespaces
  • ‚úÖ Type information is inferred from linked_schemas
  • ‚úÖ Location structure is rebuilt in Schema.org format
  • ‚úÖ Tags are mapped to appropriate namespaced properties
  • ‚ùå Custom fields not in the base Murmurations schema cannot be recovered
  • ‚ùå Specialized metadata unique to the original unified profile is permanently lost

Recovery Strategy

Our approach to handling this data loss:

  1. Source of Truth: We maintain the complete unified profiles as our source of truth
  2. Derived Profiles: The Murmurations profiles are treated as derived, simplified versions
  3. Smart Recovery: Our web app uses custom logic in convertSchema() to recover predictable information
  4. Inference: We use the available Murmurations data to infer and reconstruct some lost fields
  5. Acceptance: We acknowledge that some specialized information cannot be recovered after conversion

This approach balances interoperability with the Murmurations ecosystem while maintaining our richer unified schema for visualization and analysis purposes.

Registering Unified Schemas with Murmurations üöÄ

To fully register our unified schemas with Murmurations, we need to:

  1. Submit schemas via GitHub PR:

    • Fork the Murmurations library repo
    • Create a new branch including our schema name
    • Add our schema files (regen-person-schema-v1.0.0.json and regen-organization-schema-v1.0.0.json) to the /schemas/ directory
    • Create a PR to the test branch of the upstream repo
    • Wait for review and approval

  2. After approval and merging to the test branch:

    • Our schemas will be available in the test environment
    • We can test creating profiles based on these schemas

  3. Update our profiles to use both schemas:

    "linked_schemas": ["people_schema-v0.1.0", "regen-person-schema-v1.0.0"]

  4. Re-upload our updated profiles:

    • Use the upload-profiles.js script (after modifying it to use our new schemas)
    • Or manually update the profiles in our GitHub repo

Dual Schema Discovery Benefit ‚ú®

When our unified schemas are registered and our profiles include both base and unified schemas in their linked_schemas array:

  • Broader Discovery: Profiles will be discoverable by BOTH queries for base schemas AND queries for our unified schemas
  • Enhanced Data: Applications using our unified schemas will get richer data with regenerative-specific fields
  • Base Compatibility: Applications using only base Murmurations schemas will still discover and use our profiles
  • No Duplication: A single profile can serve both ecosystem needs without maintaining separate versions

This approach creates a "superset" relationship where our unified schemas extend the base schemas with additional fields while maintaining full compatibility and discoverability.

The Regen Mapping project includes several scripts for schema conversion, profile management, and integration with Murmurations.

Schema Conversion

# Convert between schema formats
node scripts/convert-schema.js murm-to-unified input.json output.json
node scripts/convert-schema.js unified-to-murm input.json output.json

# Test Cambria transformations
node scripts/test-cambria-transformations.js

Profile Management

# Standard conversion (without @reverse links)
node scripts/convert-unified-to-murmurations.js

# Lossless conversion with @reverse links
node scripts/lossless-conversion.js convert-all

# Test lossless round-trip conversion
node scripts/test-lossless.js

# Upload profiles to Murmurations
node scripts/upload-profiles.js
node scripts/upload-profiles-new.js

# Complete workflow: convert, validate, submit, and test
node scripts/update-and-publish-profiles.js
